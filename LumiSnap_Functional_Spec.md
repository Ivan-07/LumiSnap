# LumiSnap 产品核心功能规格说明 (PRD Draft)

## 1. 功能清单 (Feature List)

以下是 LumiSnap 应用所需的核心功能模块：

| 功能模块 | 描述 |
| :--- | :--- |
| **相机/拍摄界面 (Camera Interface)** | 应用的主界面。用户在这里取景、构图，是所有操作的起点。它需要支持基本的摄像头切换（前后置）、闪光灯控制。 |
| **场景模板选择器 (Scene Selector)** | 位于拍摄界面的核心交互元件。它以一个横向滚动列表或轮盘的形式存在，用户通过滑动来选择不同的"文案风格模板"，如"小红书种草"、"咸鱼卖货"、"朋友圈日常"等。 |
| **AI 生成引擎 (AI Generation Core)** | 产品的"黑盒"后台。当用户点击拍摄按钮时，它负责接收图像数据和选定的场景模板，调用多模态大模型，并将生成的文案返回给前端。 |
| **结果展示与编辑页 (Result & Edit Page)** | 拍照后跳转的页面。它会同时展示拍摄的图片和 AI 生成的配套文案。提供基础的编辑和调整功能。 |
| **导出与分享 (Export & Share)** | 在结果页的功能按钮。用户可以将生成的"图文合一"内容保存到本地相册，或直接分享到其他社交平台。 |
| **首次用户引导 (Onboarding)** | 新用户第一次打开 App 时出现的引导流程。用最简洁的方式（比如几张滑动的卡片）教会用户"选场景 -> 拍照 -> 得文案"的核心玩法。 |

## 2. 用户操作路径 (User Flow)

一个典型用户的完整使用路径如下：

**首次使用:**
1.  **打开 App** -> 触发 **首次用户引导 (Onboarding)**。
2.  引导结束，进入 **相机/拍摄界面**。
3.  系统请求相机权限。

**日常使用:**
1.  **打开 App**，直接进入 **相机/拍摄界面**。
2.  在界面上滑动 **场景模板选择器**，选择一个想要的风格（例如，选择"小红书种草"）。
3.  对准想要拍摄的物品或场景，点击 **拍摄按钮**。
4.  （后台 **AI 生成引擎** 开始工作）
5.  画面跳转至 **结果展示与编辑页**，用户看到图片和匹配的文案。
6.  **Case A (满意):** 直接点击 **导出与分享** 按钮，保存图片或分享到小红书。 -> **任务完成**。
7.  **Case B (基本满意，想微调):** 在结果页对文案进行少量手动修改，或点击"换一句"按钮（如果提供）。
8.  点击 **导出与分享** 按钮。 -> **任务完成**。

## 3. 功能点与解决的问题对齐

每个功能的设计都直接回应了我们之前分析出的用户痛点。

| 功能点 | 解决的核心问题/痛点 |
| :--- | :--- |
| **场景模板选择器** | **解决"提示词认知壁垒"**: 用户不再需要思考如何写 Prompt，只需做选择题，极大降低了 AI 使用门槛。 |
| **即拍即得的流程** | **解决"灵感稍纵即逝"和"流程繁琐"**: 将原本分散的多步操作（拍照、打开App、上传、写词、生成）合为一步，让用户在灵感迸发的瞬间完成创作。 |
| **AI 自动生成文案** | **解决"表达焦虑"和"词穷"**: 为用户提供一个高质量的文案起点，消除他们面对空白输入框的恐惧，鼓励更多分享。 |
| **风格化模板** | **解决"内容风格一致性"难题**: 帮助内容创作者快速生成符合其账号调性的文案，提升专业度和效率。 |
| **结果页的微调功能** | **给予用户最终掌控感**: AI 生成的结果可能不是 100% 完美，简单的修改功能满足了个性化需求，同时避免了重写的麻烦。 |

## 4. 功能逻辑关系与优先级

*   **核心功能 (Core Loop):**
    1.  **场景模板选择器**
    2.  **相机/拍摄界面** (核心是拍照动作)
    3.  **AI 生成引擎**
    4.  **结果展示页** (最基础的展示)
    *   **逻辑关系:** 这是产品的核心闭环，缺一不可。`选择模板` -> `拍照` -> `生成` -> `展示`。

*   **辅助功能 (Supporting Features):**
    *   **导出与分享**: 让用户的创作成果能离开 App 产生价值，是核心闭环的延伸。
    *   **编辑功能**: 提升结果的满意度和可用性。
    *   **首次用户引导**: 降低新用户上手门槛，提升留存。

*   **优先级:**
    1.  **P0 (最高):** 核心功能 (Core Loop) 必须全部实现。
    2.  **P1 (次高):** 导出与分享 (必须有"保存到相册"和"复制文案")，首次用户引导。
    3.  **P2 (中等):** 基础的文本编辑功能。

## 5. MVP (最小可用产品) 视角收敛

为了最快地验证核心价值（"拍张照就能得到想要的文案"），MVP 版本应聚焦于核心闭环，剥离一切非必要功能。

#### **MVP 应保留的功能:**

*   ✅ **相机界面**: 功能极简，能拍照即可。
*   ✅ **场景模板选择器**:
    *   **内容:** 提供 **3-5 个** 最核心、差异化最明显的模板（例如：**小红书种草、朋友圈日常、咸鱼卖货**）。
    *   **形式:** 简单的横向滚动列表。
*   ✅ **AI 生成引擎**: 产品的技术核心，必须稳定可用。
*   ✅ **结果展示页**:
    *   只展示图片和生成的文案。
    *   提供两个按钮：【**保存图片**】和【**复制文案**】。
*   ✅ **首次用户引导**: 可以是启动时的一张静态图片说明，解释核心玩法。

#### **MVP 可放弃的功能:**

*   ❌ **高级编辑功能**: 如"智能调节滑竿"、重新生成等，全部砍掉。只保留最基础的文本框手动修改。
*   ❌ **复杂的分享功能**: 不用做"分享到微信/微博"等 API 对接，让用户先保存/复制，再手动去分享。
*   ❌ **用户账户系统**: 无需登录注册。
*   ❌ **历史记录/作品库**: 本地不保存任何历史记录。
*   ❌ **设置页面**: 所有可配置项全部移除。
*   ❌ **模板市场**: 这是后续生态的核心，但 MVP 阶段完全不需要。
*   ❌ **前后置摄像头切换/闪光灯**: 除非特定场景需要，否则可以先只支持后置摄像头。
